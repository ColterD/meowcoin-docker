 * Cross-link to README.md and scripts/bulk-edit-examples.md.

## Code Style
- Follow project ESLint and Prettier rules.
- Use region folding markers and JSDoc for all major modules.

## PR Process
- Fork, branch, and submit pull requests with clear descriptions.
- Reference relevant roadmap items and TODOs.

## Testing
- Add/expand tests for all new features and bugfixes.
- Run `npm test` and ensure CI passes before submitting.

## Bulk Editing
- Use VS Code multi-cursor, Find & Replace, or CLI tools for large changes.
- See [bulk-edit-examples.md](scripts/bulk-edit-examples.md) for tips.

## Best Practices
- Commit in logical, reviewable steps.
- Use TODO[roadmap] for all roadmap-related todos.
